4. Data Storage and Location

4.1 Local Storage

Your data is stored locally on your device using:

  • SQLite Database: Encrypted local database for app data
  • Secure Storage: Encrypted storage for sensitive information (using FlutterSecureStorage)

4.2 Cloud Storage

We use Supabase as our cloud storage provider to enable data synchronization across your devices.

Data Location: Your data is stored on Supabase servers located in North America (United States). This ensures compliance with data residency requirements for users in the United States and Canada.

Data Security: All data transmitted to and stored in Supabase is encrypted using industry-standard encryption protocols (HTTPS/TLS).

5. Data Sharing and Third-Party Services

5.1 Supabase

We share your data with Supabase solely for the purpose of:

  • Data storage and backup
  • Data synchronization across devices
  • User authentication

Supabase is a trusted service provider that adheres to strict data security standards and privacy practices. Supabase does not use your data for their own purposes.

For more information about Supabase's privacy practices, visit: https://supabase.com/privacy

5.2 Local Notification Services

  • Purpose: Sending task reminders and notifications
  • Data Sharing: Does not share any data, runs only on local device

5.3 No Other Third-Party Sharing

We do NOT:

  • Sell your personal information to third parties
  • Share your data with advertisers
  • Allow third parties to access your data for their own purposes

6. Data Security

We implement industry-standard security measures to protect your data:

  1. Encryption in Transit: All data transmitted between your device and our servers uses HTTPS/TLS encryption
  2. Encryption at Rest: Sensitive data is encrypted when stored locally and in the cloud
  3. Secure Authentication: User authentication is handled securely through Supabase Auth
  4. Access Controls: Data is isolated per user, and users can only access their own data
  5. Regular Backups: Cloud data is regularly backed up to prevent data loss
  6. Security Audits: Regular security audits and vulnerability scans
